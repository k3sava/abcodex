Simon Willison's May 14 post, tagged across ai, react, generative-ai, ai-assisted-programming, and coding-agents, picks up Mitchell Hashimoto's observation that programming languages and frameworks no longer lock you in the way they once did. The argument is that AI-assisted coding has driven switching costs toward near-zero: translating a codebase, refactoring a stack, or rebuilding in a different framework is now cheap enough that the lock-in calculus has shifted. Simon's tagging of React alongside general AI tags suggests he's applying this specifically to front-end framework choices — the kind of decision that used to carry years of sunk cost.

For substrate: if framework switching costs drop significantly, the case for betting the architecture on any single framework weakens. Decisions that feel permanent are now closer to reversible. For career-ops: this is a concrete argument against deep framework specialization as a career moat — useful framing for any conversation Kesava has about what technical skills retain value as AI coding matures.
