# Steve Krouse

## Bio
Steve Krouse built Val Town as a platform where developers write and deploy serverless JavaScript functions in the browser, with no build step, no deploy pipeline. His central insight has been that friction in developer tooling compounds: every step between "idea" and "running code" costs adoption. Val Town reached 90,000 users in May 2026, with 25% of new users that month reporting they discovered the platform via Claude. That figure surfaced a new acquisition mechanism: AI coding agents that generate code referencing a platform do the discovery work that landing pages and ads traditionally do, inside the developer's flow.

## Operating themes
- **Developer acquisition via AI agents:** Platforms that AI agents naturally reach for when solving relevant problems acquire users at the moment of task completion, without traditional funnel mechanics.
- **Reducing developer friction:** The fewer steps between idea and deployed code, the higher the adoption rate. Val Town's model is zero-config serverless JS in the browser.
- **AI-native product loops:** Shipping an AI coding agent (Townie) on top of Val Town creates a feedback loop where the product and the agent that recommends the product reinforce each other.
