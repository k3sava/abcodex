The AEO triangle, presence, relevance, manual-action propagation

Convergence

Six operators converge on a coherent model for Answer Engine Optimization: measure on three layers (presence, readiness, impact), engineer at the passage level, treat AEO as a GTM capability not an SEO tweak, and watch out for ghost-citation gaps + manual-action propagation + slop-loop self-confirmation. Together these form a triangle: measurement, content shape, and integrity.

Variation

  • Solis provides the metric layer.
  • Indig provides the gap diagnosis (citation vs mention).
  • King provides the content unit (passage).
  • Voje provides the org placement (GTM, not SEO).
  • Natividad provides the content shape (POV beats coverage).
  • Gabe and Ray provide the integrity warnings (manual actions propagate; slop self-confirms).
  • Convergence: AEO is multi-layered, passage-grain, and integrity-sensitive. Treating it as "SEO with new keywords" misses every dimension.

Implication

Stand up AEO as a named GTM capability with the three-layer dashboard (Solis), passage-grain content briefs (King + Natividad), citation/mention tracking (Indig), and an integrity watch on manual actions + slop loops (Gabe + Ray). Move it out of SEO; staff it like a launch surface.
